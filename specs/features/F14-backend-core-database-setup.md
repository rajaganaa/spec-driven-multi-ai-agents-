# F14: Backend Core & Database Setup

## Goal
Establish the foundational FastAPI backend and PostgreSQL database connection, defining initial data models for FAQs.

## Acceptance Criteria
- [ ] A FastAPI application initializes successfully.
- [ ] A connection to a PostgreSQL database is established and verified.
- [ ] The initial database schema for the 'faqs' table (including fields like question, answer, category) is defined and migrated.
- [ ] A basic '/health' endpoint returns a 200 OK status.
- [ ] No logging of sensitive request data or user input is enabled by default in the core application setup.
